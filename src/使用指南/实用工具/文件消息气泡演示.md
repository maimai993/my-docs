## 组件特点总结

### 布局差异

**支持预览的文件（图片、视频）：**
- 预览内容显示在上方
- 文件名称和大小在同一行显示在下方
- 右下角固定下载按钮

**不支持预览的文件：**
- 文件图标显示在左侧
- 文件名称在上方
- 文件大小在下方
- 右下角固定下载按钮

### 交互效果

- 鼠标悬停时卡片有轻微上浮效果
- 下载按钮悬停时有颜色变化
- 滚动进入视口时有淡入动画
- 加载状态显示加载动画
- 错误状态显示错误提示

### 响应式设计

- 桌面端：最大宽度 320px
- 移动端：最大宽度 280px
- 图片/视频预览最大高度自动调整

## 使用建议

1. **图片文件**：建议提供独立的预览链接，使用较低分辨率的图片作为预览，提高加载速度
2. **视频文件**：确保视频格式兼容主流浏览器（MP4、WebM）
3. **文件大小**：组件会自动格式化为合适的单位（B、KB、MB、GB）
4. **文件类型**：正确设置文件类型以获得最佳图标和布局

## 代码示例

```vue
<template>
  <div>
    <!-- 图片文件 -->
    <FileMessage
      fileName="示例图片.jpg"
      :fileSize="2048576"
      downloadUrl="https://example.com/file.jpg"
      previewUrl="https://example.com/preview.jpg"
      fileType="image"
    />
    
    <!-- 视频文件 -->
    <FileMessage
      fileName="演示视频.mp4"
      :fileSize="15728640"
      downloadUrl="https://example.com/video.mp4"
      fileType="video"
    />
    
    <!-- 文档文件 -->
    <FileMessage
      fileName="项目文档.pdf"
      :fileSize="1048576"
      downloadUrl="https://example.com/document.pdf"
      fileType="document"
    />
  </div>
</template>

<script>
import FileMessage from '../components/FileMessage.vue'

export default {
  components: {
    FileMessage
  }
}
</script>
```

这个组件可以很好地集成到聊天界面中，提供一致的用户体验。
